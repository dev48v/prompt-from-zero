🕸️ Graph of Thoughts — Branch, Merge & Refine Thoughts in a Graph

A tree can only branch. A graph can branch, then merge the best pieces of several branches into one.

Tree of Thoughts explores many lines of reasoning and picks the single best one — but a winning branch is often partly right in one place and another branch is right somewhere else. Graph of Thoughts (GoT) makes each thought a node in a graph, so besides branching you can aggregate — feed several thoughts into one merge node that keeps the good pieces of each — and refine — loop a node back on itself to improve it. The task below is merge-sort by thought: split a list, sort the halves as separate branches (each with a lingering bug), then aggregate the two sorted halves into one, and refine the merge until it is genuinely fully sorted — a result no single branch reached. 🔨 Build a Graph of Thoughts pipeline

    
Model the reasoning as a graph of operations (a "GoO"): nodes are thoughts, and a small controller applies four operations — generate (branch), score (rank), aggregate (merge several thoughts into one), and refine (loop a thought to improve it) — until a thought crosses a quality bar. The whole trick over Tree of Thoughts is the aggregate edge: it lets the good pieces of different branches combine, so the final answer can beat every branch it was built from.

    // A GoT graph is nodes (thoughts) + edges (operations).
// Each thought carries its content and how good it is.
let nextId = 0;
function thought(content, parents = [], op = "generate") {
  return { id: nextId++, content, parents, op, score: null };
}

// The graph is just the set of thoughts; edges live in `parents`.

    generate — branch into several candidate thoughts
    // Same first move as Tree of Thoughts: produce more than one
// next thought so the search has diverse material to work with.
async function generate(node, n = 3) {
  const out = await llm(
`Thought so far: ${node.content}
Propose ${n} DIFFERENT next thoughts toward the goal, one per line.`);
  return out.split("\n")
    .map(line => thought(line, [node.id], "generate"));
}
// For a decomposable task, "generate" can also be a SPLIT:
// break the input into sub-thoughts solved on separate branches.
    3. 

      
  3. 3
    score — rate each thought so you can rank them
    // A scorer turns a thought into a number. Objective is best:
// a validator, a test, a metric. Here: how sorted is a list?
function score(node) {
  const xs = node.content;                 // e.g. [1,3,2,4]
  let inOrder = 0;
  for (let i = 1; i < xs.length; i++)
    if (xs[i - 1] <= xs[i]) inOrder++;
  node.score = inOrder / (xs.length - 1);  // 1.0 = fully sorted
  return node.score;
}
// No hard metric? Ask the model: "Rate 0-10 how good this is."

  4. 4
    aggregate — merge several thoughts into ONE (the key move)
    // This is what a tree can't do. Take multiple thoughts and
// COMBINE their good parts into a single, better thought.
function aggregate(nodes) {
  // e.g. merge two sorted halves into one sorted list
  const merged = mergeSortedLists(nodes.map(n => n.content));
  const t = thought(merged, nodes.map(n => n.id), "aggregate");
  score(t);
  return t;
}
// With an LLM: "Here are 3 partial answers. Write ONE answer that
// keeps the best of each and fixes their mistakes."
    5. 

      
  5. 5
    refine — loop a thought back on itself to improve it
    // A self-loop edge: feed a thought back in to polish it.
// Repeat until the score stops improving or hits the bar.
async function refine(node, maxLoops = 3) {
  let cur = node;
  for (let i = 0; i < maxLoops; i++) {
    const improved = await llm(
`Improve this, fixing any remaining errors:
${cur.content}`);
    const t = thought(improved, [cur.id], "refine");
    if (score(t) <= cur.score) break;   // no gain -> stop
    cur = t;
  }
  return cur;
}

    Write the controller — the graph of operations (GoO)
    // The controller executes a plan of operations over the graph.
// This plan says: split, sort each half, MERGE them, then refine.
async function got(problem) {
  const [a, b] = split(problem);          // generate / branch
  const left  = await refine(await sortBranch(a));
  const right = await refine(await sortBranch(b));
  const merged = aggregate([left, right]); // <-- the graph move
  const final  = await refine(merged);      // polish the merge
  return final;
}
// The plan itself is data — you can reorder or reuse operations.

  7. 7
    Keep the best & stop — a quality bar with a budget cap
    // GoT calls the model a LOT (generate + score + aggregate +
// refine). Bound it: keep only the top-k thoughts per stage,
// and stop the moment a thought clears the bar.
function keepBest(nodes, k = 3) {
  return nodes.sort((x, y) => y.score - x.score).slice(0, k);
}
const CONFIG = { branches: 3, keep: 3, refineLoops: 3, maxNodes: 40 };
// stop when score >= 1.0 (fully sorted) OR node budget spent.
