🪞 Reflexion — Learn From Your Own Mistakes, Then Retry

Fail once, then keep failing the same way — unless you write the lesson down first.

A one-shot agent that gets a task wrong has no idea why — so when you simply tell it to "try again", it confidently repeats the very same mistake. Reflexion adds a memory of its own errors: after an attempt is marked FAILED by an evaluator (tests, a self-critique, a checker), the agent writes a short verbal reflection — "I assumed X; the failing case shows Y; next time I should Z" — and stores it. The next attempt sees that lesson in its context and stops repeating the error. Pick a task and watch the trial → evaluate → reflect → retry loop climb to success:

🔨 Build a Reflexion loop

    
Take a task with a checkable signal — unit tests, a validator, or a self-critique. Then loop: generate an attempt, evaluate it, and if it fails, ask for a short reflection on why, append that lesson to a running reflection memory, and retry with the memory in context — until it passes or you hit a try cap. You end up with a correct solution and a readable log of the lessons it taught itself along the way.

    
    
      
  1. 1
    Start with a task that has a checkable signal
    // Reflexion needs a way to know an attempt FAILED. The cleanest
// signal is automated: unit tests, a validator, an exact match.
const task = "Write isPalindrome(s): true if s reads the same " +
             "forwards and backwards, ignoring case and spaces.";

const tests = [
  ["racecar", true], ["RaceCar", true],
  ["a man a plan a canal panama", true], ["hello", false],
];
// The tests are the evaluator. No tests? Use a self-critique (step 3).
    2. 

      
  2. 2
    Generate attempt 1 (no memory yet)
    // First pass has an empty reflection memory — it's a plain
// one-shot attempt. We expect it might miss an edge case.
let memory = [];                    // verbal lessons accumulate here

function buildPrompt(task, memory) {
  const lessons = memory.length
    ? "\nLessons from past attempts:\n- " + memory.join("\n- ")
    : "";
  return `Task: ${task}${lessons}\nWrite the solution:`;
}

let code = await llm(buildPrompt(task, memory));   // attempt 1
    3. 

      
  3. 3
    Evaluate it — get a pass/fail + concrete feedback
    // The evaluator returns BOTH a verdict and feedback the model
// can learn from — a failing test case, a stack trace, a critique.
function evaluate(code, tests) {
  const fn = compile(code);
  for (const [input, want] of tests) {
    const got = fn(input);
    if (got !== want)
      return { pass:false,
        feedback:`fn(${JSON.stringify(input)}) returned ` +
                 `${got}, expected ${want}` };
  }
  return { pass:true };
}
// No tests? Ask the model itself: "Critique this solution. Any bug?"
    4. 

      
  4. 4
    On failure, prompt for a verbal reflection
    // This is the heart of Reflexion: turn the failure into a short,
// written LESSON — "I assumed X; case Y shows it's wrong; do Z."
async function reflect(task, code, feedback) {
  return llm(
`Task: ${task}
My attempt:
${code}
It FAILED this check: ${feedback}

In ONE or two sentences, write a lesson for my next attempt.
Be concrete: what did I assume, what does the failure prove,
what should I do differently? Do NOT rewrite the code.`);
}
// e.g. -> "I forgot to strip spaces before comparing; normalise
//          the string (lowercase + remove non-letters) first."
    5. 

      
  5. 5
    Append the lesson to reflection memory
    // Memory is just a growing list of verbal lessons. Nothing in the
// model's weights changes — the "learning" lives in the context.
const lesson = await reflect(task, code, result.feedback);
memory.push(lesson);

// memory now reads like a notebook of self-taught rules:
//   1. Strip spaces and lowercase before comparing.
//   2. Handle the empty string as a valid palindrome.
// Keep it short — old, stale lessons can be trimmed if it grows.
    6. 

      
  6. 6
    Retry with the memory in context
    // Re-run generate, but now buildPrompt() injects every lesson.
// The model reads its own past mistakes and avoids repeating them.
code = await llm(buildPrompt(task, memory));   // attempt 2
const retry = evaluate(code, tests);

// Prompt the model now sees:
//   Task: Write isPalindrome(s) ...
//   Lessons from past attempts:
//   - Strip spaces and lowercase before comparing.
//   Write the solution:
// -> attempt 2 normalises the string first, and passes.
    7. 

      
  7. 7
    Wrap the whole loop with a try cap
    // generate -> evaluate -> (fail) reflect -> append -> retry,
// stopping the instant it passes OR we hit maxTries.
async function reflexion(task, tests, maxTries = 4) {
  let memory = [];
  for (let attempt = 1; attempt <= maxTries; attempt++) {
    const code = await llm(buildPrompt(task, memory));
    const result = evaluate(code, tests);
    if (result.pass) return { code, memory, attempt };  // success
    const lesson = await reflect(task, code, result.feedback);
    memory.push(lesson);                                // learn
  }
  return { code:null, memory, failed:true };   // gave up cleanly
}
    8. 

      
  8. 8
    Use it where there's a real signal — and watch the evaluator
    // Reflexion is only as good as its evaluator. A weak or wrong
// checker breeds confident-but-wrong reflections, so prefer an
// objective signal (tests, a validator) over pure self-critique.
const { code, memory, attempt, failed } = await reflexion(task, tests);
if (!failed) console.log(`Passed on attempt ${attempt}`);
console.log("Lessons it taught itself:\n- " + memory.join("\n- "));

// ✅ Great for coding, agent tasks, anything you can auto-check.
// ⚠️ With no checkable signal, or a bad one, it can over-correct
//    or thrash — cap the tries and keep the reflections short.
    9.
