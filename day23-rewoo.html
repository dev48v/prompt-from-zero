🧩 ReWOO — Reasoning WithOut Observation

Same answer — but ReWOO calls the model only twice.

A multi-hop question needs several tool lookups chained together. ReAct calls the LLM once per step: think, act, read the observation, think again — the model is invoked every single cycle. ReWOO calls the LLM once to write the entire plan of tool calls up front (using #E1, #E2… placeholders that feed each other), a plain Worker runs the tools with no model in the loop, then one final Solver call turns the evidence into an answer. Press ▶ and watch the LLM-call counters diverge. It's a real deterministic run over a fixed knowledge set — no live model.

Question: "What is the population of the capital of the country that won the 2018 World Cup?"
Speed 5x
🔁 ReAct — interleaved
Think → Act → Observe, calling the model every step.
LLM calls: 0
tool runs: 0
🧩 ReWOO — plan, work, solve
One Plan call, tools run with no model, one Solve call.
LLM calls: 0
tool runs: 0

Both reach the same final answer. The chain here is 3 hops (winner → capital → population), so ReAct makes 3 reasoning LLM calls + 1 final = 4, while ReWOO always makes exactly 2 (plan + solve) no matter how many hops — the tool runs are cheap non-model calls. Add hops and ReAct's counter climbs; ReWOO's stays flat at 2. The catch: ReWOO commits to the plan blind, so it can't react if a tool returns a surprise — that's the tradeoff you'll see in UNDERSTAND.

🔨 Build it yourself

    
ReWOO is three parts and only two of them touch the model: a Planner prompt that emits the whole variable-linked plan, a plain Worker loop that runs the tools and substitutes #E1→result, and a Solver prompt that turns the collected evidence into the final answer.

    
    
      
  1. 
        
    1
    Planner call — write the entire plan up front
    
        
    One LLM call. Give the model the question and the list of tools, and ask it for a numbered plan where each step assigns a result variable (#E1, #E2…) and may reference earlier ones. Crucially the model never sees a tool result here — it reasons purely from the question, so this is "reasoning without observation".
    
        
    const plannerPrompt = `Question: "${question}"
Tools: Search[query], Population[city]
Write a plan. For each step output:
  Plan: <reasoning>
  #E<n> = Tool[input, may cite #E1..]
Do NOT solve yet — only plan.`;
const plan = await llm(plannerPrompt);   // LLM call #1
    
      
    2. 
      
  2. 
        
    2
    Parse the plan into steps
    
        
    Turn the text plan into structured steps: variable name, tool, and raw input. A regex per line is enough. You now have an ordered list you can execute — the model's job is done until the very end.
    
        
    const steps = [];
for (const line of plan.split("\n")) {
  const m = line.match(/^#(E\d+)\s*=\s*(\w+)\[(.+)\]$/);
  if (m) steps.push({ id: "#"+m[1], tool: m[2], input: m[3] });
}
// [{id:"#E1", tool:"Search", input:"2018 World Cup winner"}, ...]
    
      
    3. 
      
  3. 
        
    3
    Worker — run the tools, substitute variables (no LLM)
    
        
    Walk the steps in order. Before calling a tool, replace any #E1 token in its input with the stored result of that earlier step. Store each result under its variable. This whole loop is ordinary code — zero model calls — which is exactly why ReWOO is cheap.
    
        
    const evidence = {};
for (const s of steps) {
  let input = s.input;
  for (const [id, val] of Object.entries(evidence))
    input = input.replaceAll(id, val);        // #E1 → its result
  evidence[s.id] = await TOOLS[s.tool](input); // real tool, no LLM
}
    
      
    4. 
      
  4. 
        
    4
    Solver call — combine evidence into the answer
    
        
    The second and final LLM call. Hand the model the original question plus the plan-with-evidence-filled-in and ask for the answer. Two LLM calls total, regardless of how many tool hops the plan had — that's the constant-call-count win over ReAct.
    
        
    let filled = plan;
for (const [id, val] of Object.entries(evidence))
  filled = filled.replaceAll(id, val);
const answer = await llm(`Question: "${question}"
Plan & evidence:\n${filled}
Now give the final answer.`);   // LLM call #2 (last)
return answer;
    
      
    5. 
      
  5. 
        
    5
    Guard: fall back to ReAct when the path is uncertain
    
        
    ReWOO's blind plan is a liability when a tool result might change what you do next — a dead end, an ambiguous entity, an error. For those, keep ReAct (or a hybrid: plan, but re-plan if the Worker hits a surprise). Route by how predictable the tool chain is.
    
        
    if (pathIsPredictable(question))       // fixed multi-hop lookup
  return reWOO(question);              // 2 LLM calls, cheap
else
  return reAct(question);              // adapt step-by-step
