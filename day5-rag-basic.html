🔨 Build it yourself

    
RAG in four moves. The retrieval half is TechFromZero Day 45 (pgvector); this is the prompt half.

    1
    Chunk your documents
    
        
    Split docs into ~200–500 token pieces. Too big = noisy context; too small = lost meaning. Embed each chunk and store the vector (Day 45).
    
      
    2. 
      
  2. 
        
    2
    Retrieve top-k for the question
    

    const hits = await search(question, { k: 3 });   // vector similarity
    
      
    3. 
      
  3. 
        
    3
    Stuff them into a grounded prompt
    

    const prompt = `Answer using ONLY the context below.
If the answer isn't there, say "I don't know."

Context:
${hits.map(h => "- " + h.text).join("\n")}

Question: ${question}`;
    
      
    4. 
      
  4. 
        
    4
    Cite + refuse
    
        
    Ask the model to quote which chunk it used, and to say "I don't know" when the context is empty. That single instruction is what kills most hallucinations.
