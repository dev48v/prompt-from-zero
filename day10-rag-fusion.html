🔀 RAG-Fusion — Ask It Many Ways, Fuse the Ranks

One query is one guess. Several queries vote.

A single phrasing can miss the right doc. RAG-Fusion has the LLM rewrite your question several ways, retrieves for each, then fuses the lists with Reciprocal Rank Fusion — docs that rank well across many phrasings float to the top. Pick a question:

① LLM rewrites → 3 retrievals
② Reciprocal Rank Fusion → final

↑ The green doc is the true answer. It may not top any single list — but appearing across several pushes its fused score above a doc that only one phrasing liked.

    Ask the LLM for query variations
    const queries = await llm(`Generate 4 different search queries for:
"${userQuery}". Vary the wording and angle. One per line.`);

  2. 2
    Retrieve a ranked list for each
    const lists = await Promise.all(
  queries.map(q => vectorSearch(q, { k: 10 }))
);
    3. 
      
  3. 3
    Reciprocal Rank Fusion (the magic)
    const scores = {};
for (const list of lists)
  list.forEach((doc, rank) => {
    scores[doc.id] = (scores[doc.id] || 0) + 1 / (60 + rank);
  });   // rank is 0-based; 60 is the standard RRF constant k
    4. 
      
  4. 4
    Sort by fused score, keep the top few
    const fused = Object.entries(scores)
  .sort((a,b) => b[1] - a[1])
  .slice(0, 4)
  .map(([id]) => byId[id]);

  5. 5
