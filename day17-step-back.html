🪜 Step-Back Prompting — Ask the Bigger Question First

Charge straight at a tricky question and you miss the forest for the trees.

A detail-heavy question tempts the model to grab at the numbers and guess. Answering directly latches onto specifics and skips the principle that governs them — so it fumbles. Stepping back first asks a more general question ("what rule actually applies here?"), recalls that principle, and only then plugs in the specifics — so it lands the right framework and the right answer. Pick a question and watch both paths run:

❓ The specific question
Ⓐ Direct — answer the specific question now
Ⓑ Step-Back — ask the general question first

🔨 Build a two-step step-back chain

    
Take a hard, specific question. First prompt for a more general "step-back" question. Answer that to surface the governing principle. Then answer the original question, handing the principle back as context. Two cheap calls turn a blind guess into grounded reasoning.

    
    
      
  1. 1
    Start with the hard, specific question
    // A question with lots of concrete detail is exactly the kind
// that tempts a model to grab the numbers and guess.
const question =
  "An ideal gas is at 2 atm. If its temperature triples and its " +
  "volume is halved, what is the new pressure?";

// Direct answering tends to mash the numbers together and miss
// the law that actually relates them. We will step back first.
    2. 

      
  2. 2
    Prompt for the step-back question
    // Ask for ONE level of abstraction up — the general principle
// or concept this specific question is an instance of.
const stepBackPrompt = `
You are about to answer a hard, specific question.
First, do NOT answer it. Instead, write one more GENERAL question
whose answer is the principle/concept needed to solve it.

Specific question: "${question}"
Step-back question:`;

const stepBackQ = await llm(stepBackPrompt);
// → "What law relates the pressure, volume and temperature
//    of an ideal gas?"
    3. 

      
  3. 3
    Answer the step-back question → the principle
    // Answering the GENERAL question is easy and reliable — it pulls
// up clean, high-level knowledge with no specifics to trip on.
const principlePrompt = `Answer concisely and precisely:
${stepBackQ}`;

const principle = await llm(principlePrompt);
// → "The ideal gas law: PV = nRT. For a fixed amount of gas,
//    P1·V1 / T1 = P2·V2 / T2 (combined gas law)."
    4. 

      
  4. 4
    Answer the original USING the principle
    // Now feed the principle back as context and ask the original
// question. The model reasons FROM the rule, not from a guess.
const finalPrompt = `
Use this principle to reason step by step, then give the answer.

Principle:
${principle}

Question:
${question}`;

const answer = await llm(finalPrompt);
// P2 = P1 · (V1/V2) · (T2/T1) = 2 · 2 · 3 = 12 atm  ✓
    5. 

      
  5. 5
    Wrap it as one reusable function
    // The whole pattern is just: abstract → recall → apply.
async function stepBack(question) {
  const stepBackQ = await llm(
    `Write one more GENERAL question whose answer is the ` +
    `principle needed to solve:\n"${question}"\nGeneral question:`);
  const principle = await llm(`Answer precisely:\n${stepBackQ}`);
  const answer = await llm(
    `Principle:\n${principle}\n\nUse it to answer:\n${question}`);
  return { stepBackQ, principle, answer };
}
    6. 

      
  6. 6
    Pair it with retrieval (RAG)
    // A general step-back query is also a BETTER search query: it
// matches the principle in your docs, not a narrow phrasing.
async function stepBackRAG(question) {
  const stepBackQ = await llm(
    `General question for the principle behind:\n"${question}"`);
  // retrieve on the GENERAL question — higher recall
  const docs = await vectorStore.search(stepBackQ, { k: 5 });
  return llm(`Context:\n${docs.join("\n")}\n\nAnswer:\n${question}`);
}
    7. 

      
  7. 7
    Steer it with a few-shot example
    // Show one good abstraction so the step-back question lands at
// the RIGHT altitude — not too vague, not still too specific.
const stepBackPrompt = `
Specific: "Which team did Aristotle's tutor's most famous
           student conquer Persia with?"
General:  "Who was Aristotle's most famous student, and what
           did he conquer?"

Specific: "${question}"
General:`;
// Few-shot pins the abstraction level; one example is usually enough.
    8. 

      
  8. 8
    Spend the extra call only where it pays
    // Step-back adds one or two calls + latency. Gate it: use it on
// reasoning / science / multi-hop questions; skip it on lookups.
async function answer(question) {
  if (needsReasoning(question)) {
    return (await stepBack(question)).answer;   // abstract first
  }
  return llm(question);                         // simple fact → direct
}
// ⚠️ Abstract first when the PRINCIPLE is unclear; not for
//    "what's the capital of France" — that has no forest to miss.
