🥇 Reranking — Retrieve Fast, Then Reorder Precisely

Fast search gets you a shortlist. A reranker puts the best answer at #1.

A bi-encoder embeds the query and every doc separately (fast, scalable — but coarse), so its top-6 ordering is rough and the truly best doc often lands at rank 4 or 5. A cross-encoder then reads the query and each doc together (slow, accurate) and re-scores every pair — so you retrieve a shortlist cheaply, then rerank it precisely. Pick a query and watch the best doc jump to the top:

① Retrieve — bi-encoder vector search (coarse)
② Rerank — cross-encoder re-scores each pair (precise)

🔨 Add a reranker to your retrieval pipeline

    
Retrieve a wide shortlist with cheap vector search, re-score every (query, doc) pair with a cross-encoder, sort, keep the top few, then feed only those to generation.

    
    
      
  1. 1
    Retrieve a wide shortlist with the bi-encoder
    const q = "How do I cancel my subscription?";

// Stage 1 — fast & cheap. Embed the query, ANN-search the
// pre-embedded corpus. Retrieve MANY (over-fetch), e.g. 50,
// not 5 — recall now, precision later.
const qVec       = await embed(q);
const candidates = await vectorStore.search(qVec, { k: 50 });
// candidates are roughly relevant, but coarsely ordered.
    2. 

      
  2. 2
    Score every (query, doc) PAIR with a cross-encoder
    // Stage 2 — slow but accurate. The cross-encoder reads the
// query AND the doc TOGETHER and returns one relevance score.
// We only run it on the ~50 candidates, never the whole corpus.
async function rerankScores(q, docs) {
  return Promise.all(
    docs.map(async d => ({
      doc:   d,
      score: await crossEncoder(q, d.text)   // joint encoding
    }))
  );
}
const scored = await rerankScores(q, candidates);
    3. 

      
  3. 3
    Sort by rerank score (high → low)
    // The reranker's order replaces the vector-search order.
// A doc that was rank 5 can now be rank 1.
scored.sort((a, b) => b.score - a.score);
    4. 

      
  4. 4
    Take the top-n — the precise final context
    // Retrieve 50 → rerank → keep just the best few.
const topDocs = scored.slice(0, 5).map(s => s.doc);
// these 5 are far cleaner than the raw top-5 from search.
    5. 

      
  5. 5
    Feed only the reranked top-n to generation
    const answer = await llm(`Answer using ONLY these documents.
If they don't contain the answer, say so.

${topDocs.map(d => d.text).join("\n---\n")}

Question: ${q}`);
// the best evidence is now first → tighter, more accurate answer.
    6. 

      
  6. 6
    Use a hosted reranker (e.g. Cohere Rerank)
    // Don't want to host a cross-encoder? Call a rerank API.
const reranked = await cohere.rerank({
  model:     "rerank-english-v3.0",
  query:     q,
  documents: candidates.map(d => d.text),
  top_n:     5
});
// returns indices + relevance_score, already sorted.
const topDocs = reranked.results.map(r => candidates[r.index]);
    7. 

      
  7. 7
    Tune top-k (retrieve) vs top-n (keep)
    // k too small → the right doc never reaches the reranker.
// k too big  → reranker is slow & costs more per query.
// n too big  → noisy context dilutes the answer.
const K = 50;   // how many the bi-encoder hands over (recall)
const N = 5;    // how many survive the rerank (precision)
// rule of thumb: over-fetch generously, keep tightly.
    8. 

      
  8. 8
    Wrap it as one drop-in two-stage retriever
    async function retrieveAndRerank(q, k = 50, n = 5) {
  const cands  = await vectorStore.search(await embed(q), { k });
  const scored = await rerankScores(q, cands);     // cross-encoder
  scored.sort((a, b) => b.score - a.score);
  return scored.slice(0, n).map(s => s.doc);
}
// swap your retriever for this — generation stays the same.
