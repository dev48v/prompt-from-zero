📚 Few-Shot Prompting

Same model. Same problem. Two prompts: zero-shot vs few-shot. Watch the format match.

Classify this product review's sentiment + extract the top complaint.
❌ Zero-shot
"Classify sentiment + extract complaint."
✅ Few-shot (2 examples)
Same task + 2 worked examples first.

Same model. Same review. Different format. Few-shot output is parseable JSON; zero-shot is prose you'd have to extract from.

🔨 Build it yourself

    
Estimated time: 10 minutes. Node + free Gemini key.

    1
    Scaffold
    

    mkdir few-shot && cd few-shot
npm init -y && npm install ai @ai-sdk/google
echo "GOOGLE_GENERATIVE_AI_API_KEY=your_key" > .env
    
      
    2. 
      
  2. 
        
    2
    Write few-shot.mjs
    
        
    

    import { generateText } from "ai";
import { google } from "@ai-sdk/google";
const model = google("gemini-2.5-flash");

const FEW_SHOT = `Classify review sentiment + extract main complaint.

Review: "Battery dies in 2 hours. Camera is great though."
Output: { "sentiment": "negative", "complaint": "battery life" }

Review: "Love the screen and price. Wish it had USB-C."
Output: { "sentiment": "positive", "complaint": "missing USB-C" }

Review: "{{REVIEW}}"
Output:`;

const review = "Beautiful display but the keyboard feels mushy under heavy typing.";

const r = await generateText({
  model,
  prompt: FEW_SHOT.replace("{{REVIEW}}", review)
});

console.log(r.text);
// → { "sentiment": "mixed", "complaint": "mushy keyboard" }

    Upgrades
    
        
      
          
    • 3-5 examples is the sweet spot. More = diminishing returns + higher token cost.
      • 
          
    • For real production, use Zod schema + `generateObject()` (Day 9 — structured output)
      • 
          
    • If task changes often, swap examples programmatically based on input class
      • 
          
    • Cache the examples in a system prompt + only swap the {{REVIEW}} bit
