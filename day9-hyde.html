🪄 HyDE — Answer First, Then Search With That

A short question barely overlaps a long answer

Embedding a bare question often misses, because questions and answers use different words. HyDE asks the LLM to write a fake answer first, then searches with that — and the fake answer looks just like the real doc. Pick a question:

🪄 LLM writes a hypothetical answer (may be partly wrong — doesn't matter):
① Search with the question
② Search with the hypothetical answer (HyDE)

↑ The green doc is the true answer. The bare question scatters it; the doc-shaped hypothetical answer pulls it to the top.

One extra LLM call before the search. Everything downstream stays the same.

    
    
      
  1. 1
    Ask the LLM to write a fake answer
    const hypo = await llm(`Write a short passage that
would answer this question. Be specific.
Question: ${query}`);
    2. 
      
  2. 2
    Embed the HYPOTHETICAL answer, not the question
    const vec = await embed(hypo);   // ← the whole trick
    3. 
      
  3. 3
    Search your real index with that vector
    const docs = await vectorSearch(vec, { k: 5 });
// the real passages, retrieved via the fake one
    4. 
      
  4. 4
    Answer from the REAL docs (toss the fake)
    // the hypothetical answer was only a search probe.
return llm(`Answer using ONLY these:\n${docs}\nQ: ${query}`);
    5. 
      
  5. 5
