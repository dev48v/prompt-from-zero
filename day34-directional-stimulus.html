PromptFromZero · Day 34

🧭 Directional Stimulus Prompting — you can't fine-tune a black-box LLM, so train a tiny policy model instead: for every input it emits a short directional stimulus (a few hint keywords) that you paste beside the prompt, nudging the frozen LLM toward the output you actually want — and the policy is optimized with the LLM's own task score as its reward (Li et al., 2023)

You can't touch the big model's weights — so steer it from the outside. A small trained policy writes per-input hint keywords; drop them beside the prompt and watch the frozen LLM's answer snap toward the facts you care about.

A giant black-box LLM (an API you can't fine-tune) gives you one lever: the text you feed it. Directional Stimulus Prompting (Li et al., 2023) turns that lever into precise steering. Instead of hand-writing one prompt, you train a tiny policy model that, for each specific input, generates a short directional stimulus — a handful of instance-specific hint keywords. You concatenate those hints with the prompt and send the whole thing to the frozen LLM; the hints pull its output toward the target. The policy is trained cheaply (supervised warm-up, then reinforcement learning using the LLM's downstream task score — ROUGE, key-point coverage, a reward model — as the signal), while the LLM never changes. Below: pick a task, toggle the hint keywords the policy proposes, and compare the frozen LLM's answer without stimulus versus with it. Everything is scripted and deterministic — no live model — but faithful to DSP's policy → stimulus → frozen-LLM loop.

🔑 You never changed the model — you changed what you fed it, per input, automatically. Directional Stimulus Prompting keeps the expensive LLM completely frozen and instead trains a small, cheap policy to write instance-specific hints. Because the policy is optimized against the LLM's own downstream score, it learns which keywords actually move the output — not what merely sounds good to a human. That makes it work even when the LLM is a black-box API you have no access to fine-tune, and it beats a single hand-written prompt because the guidance is tailored to each input rather than fixed. Its two requirements: a way to score the output (so the policy has a reward) and enough data to train the little policy. Where those exist, a model you can't touch becomes a model you can precisely aim.

Policy model reads the input → emits directional stimulus (hint keywords) → hints are prepended to the prompt → frozen LLM produces a steered output. Train the policy with the LLM's task score as reward; keep the LLM frozen forever.

🔨 Build it yourself

    
Directional Stimulus Prompting is a small policy model wrapped around a frozen LLM you don't control. The policy reads each input and writes a short stimulus (hint keywords); you prepend it to the prompt and call the LLM; you score the output; and you train only the policy to maximize that score. No fine-tuning of the big model — just learning what to feed it. Here is the whole thing.

    
    
      
  1. 
        
    1
    A frozen black-box LLM + a tiny tuneable policy
    
        
    The big model is a black box — an API you call, whose weights you cannot change. The only thing you control is the text you send. So you add a second, small model (e.g. a T5-base) whose whole job is to write hints. Only this one is ever trained.
    
        
    // 1. Two models. Only the small one will learn.
async function frozenLLM(prompt) {           // black box, never fine-tuned
  return callApi(prompt);                    // you only control the text in
}
const policy = new SmallSeq2Seq("t5-base");  // ~220M params, tuneable
// the giant LLM might be 100x+ bigger AND unreachable for training.
    
      
    2. 
      
  2. 
        
    2
    Policy generates the directional stimulus (per input)
    
        
    Feed the input to the policy; it emits a short, instance-specific stimulus — hint keywords for summarization, dialogue-act + slot values for a chatbot, aspect words for controlled generation. This is not a fixed global prompt: every input gets its own tailored hints.
    
        
    // 2. stimulus = policy(input). Short, and specific to THIS input.
function makeStimulus(input) {
  return policy.generate(input);   // e.g. "Aurora X1; foldable; $2,199; March; pre-orders"
}
const stimulus = makeStimulus(article);   // a few hint keywords, not a paragraph
    
      
    3. 
      
  3. 
        
    3
    Prepend the stimulus, call the frozen LLM
    
        
    Concatenate the hints with your normal prompt and send the whole thing to the frozen LLM. The hints act as a soft constraint — the model tends to weave them into its answer, so its output moves in the direction you chose.
    
        
    // 3. Build the stimulus-augmented prompt; run the frozen LLM.
function steer(input, stimulus, task) {
  const prompt =
    `${task}\n\nHint keywords: ${stimulus}\n\nInput: ${input}\n\nAnswer:`;
  return frozenLLM(prompt);        // same model, better-aimed input
}
const steered = await steer(article, stimulus, "Summarize the article.");
    
      
    4. 
      
  4. 
        
    4
    Score the output — that score is the reward
    
        
    You need a number that says how good the steered output is: ROUGE against a reference summary, key-point coverage, task success, or a reward model. This score is the training signal for the policy — the frozen LLM's behavior becomes the reward function.
    
        
    // 4. Downstream metric on the frozen LLM's output = the reward.
function reward(steered, reference) {
  return rouge(steered, reference);      // or coverage / task-success / RM score
}
const r = reward(steered, goldSummary);  // e.g. 0.20 (no hints) -> 1.00 (good hints)
    
      
    5. 
      
  5. 
        
    5
    Train the policy: SFT warm-up, then RL on the reward
    
        
    Two stages. First supervised fine-tuning: extract pseudo-hints from references (e.g. keywords present in the gold summary) and teach the policy to imitate them — a sane starting point. Then reinforcement learning (PPO): sample stimuli, run the frozen LLM, and push the policy toward hints that earn a higher reward. The gradients flow only into the small policy.
    
        
    // 5a. SFT warm-up: imitate pseudo-hints pulled from references.
policy.fit(trainInputs, extractKeywords(references));   // cheap, stabilizing

// 5b. RL (PPO): reward = the frozen LLM's downstream score.
for (const input of trainInputs) {
  const stim   = policy.sample(input);            // explore hint variants
  const out    = await frozenLLM(buildPrompt(input, stim));
  const R      = reward(out, refFor(input));      // LLM behavior -> scalar
  policy.ppoUpdate(input, stim, R);               // ONLY the policy learns
}
// the giant LLM is never touched -> works even for API-only models.
    
      
    6. 
      
  6. 
        
    6
    Serve it — and know when it fits
    
        
    At inference the loop is trivial: policy(input) → stimulus → frozen LLM → output, with everything frozen. DSP shines when you can't fine-tune the LLM (closed API), the task is steerable by keywords/slots, and you have a scorer + some training data. If you have no reward signal or no data for the policy, fall back to a hand-written prompt or few-shot examples instead.
    
        
    // 6. Inference: freeze everything; just aim.
function serve(input) {
  const stimulus = policy.generate(input);        // trained, now frozen
  return frozenLLM(buildPrompt(input, stimulus)); // steered output
}
// use it when: LLM is black-box + task is steerable + you have a scorer.
if (!haveScorer || !haveTrainData) useHandWrittenPrompt();  // DSP needs both

🎉 Day 34 of PromptFromZero done.
You learned Directional Stimulus Prompting (DSP, Li et al., 2023): when the big LLM is a frozen black box you can't fine-tune, train a tiny policy model to emit a short, per-input directional stimulus — hint keywords — that you prepend to the prompt to steer the frozen LLM's output. The policy is warmed up with supervised learning on pseudo-hints, then optimized with reinforcement learning using the LLM's own downstream score (ROUGE, coverage, task success) as the reward. The key shift: you don't change the expensive model, you learn what to feed it, and the guidance is tailored to each input instead of a single fixed prompt. It works precisely because the LLM stays untouched — so it applies to closed APIs — and it needs two things: a scorer for the reward and data to train the policy. 👉 Next up — more ways to precisely steer a model whose weights you don't control.
