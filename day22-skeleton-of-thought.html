🦴 Skeleton of Thought — Outline First, Expand in Parallel

Same answer, delivered ~2–3× sooner — by writing all the points at once.

A normal LLM writes an answer strictly left-to-right, one token after another. Skeleton of Thought (SoT) asks the model for a quick bullet-point skeleton first, then expands every bullet in parallel as separate calls. Because the expansions run at the same time, the wall-clock finish time is the skeleton plus the slowest single point — not the sum of all of them. Press ▶ and watch the two panels race. The timing is real arithmetic from the tokens/sec slider.

Question: "How can I make my website load faster?" — Sequential writes the skeleton then every point one after another (total = sum of all). SoT writes the skeleton, then fires all point-expansions concurrently (total = skeleton + the longest single point). The speedup is real when the points are independent; it shrinks toward 1× when there are few points or one giant point.

🔨 Build it yourself

    
Two LLM stages: one call for the skeleton, then N parallel calls (one per point), stitched back together. The whole orchestration is a skeleton prompt, a parse, a Promise.all, and a join.

    
    
      
  1. 
        
    1
    Ask for a skeleton
    
        
    First prompt: get a terse, numbered outline — just 3–8 short point titles, no prose. This call is small and fast because you cap it to a few words per point. The skeleton is the plan the parallel workers will each flesh out.
    
        
    const skeletonPrompt = `You are answering: "${question}".
First give ONLY a skeleton: 3-8 numbered point titles,
<=6 words each, no explanations. One per line.`;
const skeleton = await llm(skeletonPrompt);
const points = skeleton.split("\n").filter(Boolean);
    
      
    2. 
      
  2. 
        
    2
    Expand every point in parallel
    
        
    Now fire one expansion call per point — all at once with Promise.all. Each prompt gets the question, the full skeleton (for context/coherence), and the one point it must expand in 1–2 sentences. Because the calls overlap, the wall-clock cost is the slowest single expansion, not the sum.
    
        
    const expanded = await Promise.all(points.map((p, i) =>
  llm(`Question: "${question}".
Full outline: ${skeleton}
Expand ONLY point ${i+1} ("${p}") in 1-2 sentences.`)
));   // all N requests run concurrently
    
      
    3. 
      
  3. 
        
    3
    Stitch the answer
    
        
    Join the point title with its expansion, in skeleton order, and you have the final structured answer. Because the model committed to the structure up front, the result is usually better-organised than a single free-form generation — a nice quality side-effect on top of the speed win.
    
        
    const answer = points
  .map((p, i) => `**${p}**\n${expanded[i]}`)
  .join("\n\n");
return answer;
    
      
    4. 
      
  4. 
        
    4
    Guard: only when points are independent
    
        
    SoT helps when points don't depend on each other (a listicle, a review, an overview). It hurts chained reasoning — a maths proof or a step-by-step derivation where point 3 needs point 2's result. A cheap classifier (or a rule) decides whether to use SoT or fall back to normal sequential generation.
    
        
    const independent = await llm(
  `Does answering "${question}" need step-by-step
   dependent reasoning? Answer yes or no.`);
if (independent.toLowerCase().startsWith("yes"))
  return normalGenerate(question);   // fall back
return skeletonOfThought(question);  // safe to parallelise
