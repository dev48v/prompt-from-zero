📝 HyDE — Search With a Hypothetical Answer, Not the Question

A 5-word question is a poor search vector. A full answer is a great one.

HyDE first drafts a hypothetical answer to the question — a full, detailed passage written as if it already knew the answer. It embeds that rich passage (not the short question) and searches with it. The fake answer lives near the real answer docs in vector space, so it finds better evidence — even if the draft contains errors, since it's only used to search, never to answer. Pick a question and watch both paths race:

🅐 Naive — embed the short query

🔨 Add HyDE to your retrieval pipeline

    
Draft a hypothetical answer, embed that, search with it, then answer from the REAL docs you found.

    
    
      
  1. 1
    Start with the user's short query
    const q = "How do I cancel my subscription?";
// 5 words. As a search vector this is sparse and weak —
// it barely overlaps with how the real answer is written.
    2. 
      
  2. 2
    Prompt the LLM for a hypothetical document
    async function draftHyde(q) {
  return await llm(`Write a short, detailed passage that
DIRECTLY ANSWERS the question, as if quoted from a help
article. Invent plausible specifics. Do NOT hedge.

Question: ${q}`);
}
const hypo = await draftHyde(q);
// e.g. "To cancel your subscription, open Account Settings,
//  choose Billing, then click Cancel Plan. Access continues
//  until the end of the current billing period; no refund
//  is issued for the remaining days."
    3. 
      
  3. 3
    Embed the hypothetical answer (not the query)
    // the draft may contain errors — that's fine.
// we embed it only to SEARCH, never to answer.
const searchVector = await embed(hypo);
// rich, answer-shaped text → a vector that sits close to
// the real answer docs in embedding space.
    4. 
      
  4. 4
    Vector-search the REAL corpus with that vector
    const realDocs = await vectorStore.search(searchVector, { k: 4 });
// retrieves genuine documents from your index — the
// hypothetical text is thrown away after this step.
    5. 
      
  5. 5
    (Optional) Rerank the retrieved docs
    // HyDE finds good CANDIDATES; a cross-encoder reranker
// then orders them precisely against the ORIGINAL query.
const ranked = await rerank(q, realDocs);   // q, not hypo
const top = ranked.slice(0, 3);
    6. 
      
  6. 6
    Answer using ONLY the real retrieved docs
    const answer = await llm(`Answer using ONLY these documents.
If they don't contain the answer, say so.

${top.map(d => d.text).join("\n---\n")}

Question: ${q}`);
return answer;   // grounded in real evidence, found via HyDE
    7. 
      
  7. 7
    Wrap it as one drop-in retriever
    async function hydeRetrieve(q, k = 4) {
  const hypo = await draftHyde(q);            // 1 extra LLM call
  const v    = await embed(hypo);             // embed the draft
  return vectorStore.search(v, { k });        // search real corpus
}
// swap your retriever for this — generation stays the same.
