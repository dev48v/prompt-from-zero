❓ Self-Ask — Let the Model Interview Itself

Leap to the final answer of a multi-hop question and a hidden middle fact gets skipped.

A question that hides a fact inside a fact — a multi-hop question — tempts the model to jump straight to the last hop. Answering directly guesses the final answer without ever surfacing the intermediate fact it depends on, and lands on the wrong thing. Self-ask first decides “are follow-up questions needed?”, then explicitly asks itself follow-up questions one at a time — answering each before moving on — until the intermediate facts are all on the table, and only then composes them into the final answer. Pick a question and watch both paths run:

❓ The multi-hop question
Ⓐ Direct — answer the final hop in one leap
Ⓑ Self-Ask — interview yourself, follow-up by follow-up

🔨 Build a self-ask solver

    
Take a multi-hop question. Seed the model with a few-shot example that shows the Follow-up / Intermediate answer format. Then loop: let it decide if a follow-up is needed, emit one, answer it (optionally via search), feed that answer back, and repeat — until it says “So the final answer is …”. You end up with the right answer and a clean trace of every intermediate fact.

    
    
      
  1. 1
    Start with the multi-hop question
    // A question that hides a fact inside a fact. The model is
// tempted to leap to the last hop and skip the middle one.
const question =
  "Who was U.S. president when the composer of " +
  "'Rhapsody in Blue' was born?";

// Answering directly, the model jumps to a president and
// skips two hidden hops: WHO composed it, and WHEN they were born.
    2. 

      
  2. 2
    Few-shot the self-ask format
    // One worked example teaches the structured Q&A scaffold:
// the gate, the Follow-up lines, the Intermediate answers,
// and the closing "So the final answer is".
const fewShot = `
Question: Who lived longer, Mozart or Beethoven?
Are follow-up questions needed here: Yes.
Follow-up: How long did Mozart live?
Intermediate answer: 35 years.
Follow-up: How long did Beethoven live?
Intermediate answer: 56 years.
So the final answer is: Beethoven.
`;
// The model will now copy this shape for new questions.
    3. 

      
  3. 3
    The gate — “are follow-ups needed?”
    // Build the prompt and let the model answer the gate first.
// "No" → it can answer directly; "Yes" → it owes a follow-up.
const prompt = fewShot + `
Question: ${question}
Are follow-up questions needed here:`;

const gate = await llm(prompt);     // -> "Yes."
const needsHops = /yes/i.test(gate);
// If needsHops is false, just read off the final answer.
    4. 

      
  4. 4
    Loop: emit a follow-up, then answer it
    // Generate up to the next "Follow-up:" line, stop, answer it,
// paste the intermediate answer back, and continue the trace.
let trace = prompt + " Yes.\n";
while (true) {
  const out = await llm(trace, { stop: ["Intermediate answer:",
                                        "So the final answer is"] });
  trace += out;
  if (/So the final answer is/i.test(out)) break;   // done
  const followUp = out.match(/Follow-up:(.*)/i)[1].trim();
  const inter = await answer(followUp);             // step 5
  trace += `Intermediate answer: ${inter}\n`;
}
    5. 

      
  5. 5
    Answer each follow-up via search (Self-Ask + Search)
    // Each follow-up is a small, single-fact question — perfect
// to hand to a search engine for an up-to-date, grounded answer.
async function answer(followUp) {
  const hit = await search(followUp);   // web / vector store
  // distil the top hit into one clean fact
  return llm(`Question: ${followUp}\nContext: ${hit}\n` +
             `Give just the answer:`);
}
// answer("Who composed 'Rhapsody in Blue'?") -> "George Gershwin"
// answer("When was George Gershwin born?")    -> "1898"
    6. 

      
  6. 6
    Compose the intermediate answers into the final
    // When the model emits "So the final answer is", the hops are
// all resolved and the last hop reads off the prior facts.
// trace:
//   Follow-up: Who composed 'Rhapsody in Blue'?
//   Intermediate answer: George Gershwin.
//   Follow-up: When was George Gershwin born?
//   Intermediate answer: 1898.
//   Follow-up: Who was U.S. president in 1898?
//   Intermediate answer: William McKinley.
//   So the final answer is: William McKinley.
const final = trace.match(/So the final answer is:?\s*(.+)/i)[1];
    7. 

      
  7. 7
    Wrap it as one reusable function
    // decide -> ask follow-up -> answer -> repeat -> final.
async function selfAsk(question, { useSearch = true } = {}) {
  let trace = fewShot + `\nQuestion: ${question}\n` +
              `Are follow-up questions needed here:`;
  for (let i = 0; i < 8; i++) {            // hop cap = safety net
    const out = await llm(trace, { stop: ["Intermediate answer:",
                                          "So the final answer is"] });
    trace += out;
    if (/So the final answer is/i.test(out)) break;
    const fu = (out.match(/Follow-up:(.*)/i) || [])[1];
    if (!fu) break;
    trace += `Intermediate answer: ${useSearch
      ? await answer(fu.trim()) : await llm(fu.trim())}\n`;
  }
  return { trace, answer: (trace.match(
    /So the final answer is:?\s*(.+)/i) || [])[1] };
}
    8. 

      
  8. 8
    Spend the follow-ups only where they pay
    // Self-ask costs one call per follow-up (plus a search each).
// The built-in gate already handles this: a single-hop question
// answers "No" and exits with zero follow-ups.
async function solve(question) {
  const { trace, answer } = await selfAsk(question);
  // inspect the trace — every intermediate fact is visible,
  // so a wrong final answer points straight at the bad hop.
  return answer;
}
// ⚠️ Worth it on compositional / multi-hop factual questions;
//    a one-hop lookup just gates out and costs nothing extra.
