PromptFromZero · Day 32

🗣️ Multi-Agent Debate — instead of trusting one model's first confident answer, spin up several independent instances, let each read and critique the others, and revise over a few rounds; idiosyncratic errors cancel, agreement tracks correctness, and the group converges on an answer more reliable than any single pass (Du et al., 2023)

One model answers a trick question with total confidence — and is wrong. Put several in a room and make them argue, and the group finds the answer.

Ask a single model a deceptively simple puzzle and it often blurts out a confident, wrong answer — there is no second opinion to catch the slip. Multi-Agent Debate (Du et al., 2023) attacks this: run several independent model instances on the same question, then over a few rounds show each agent the others' answers and reasoning and ask it to critique and revise. Because different instances tend to make different mistakes, a well-argued correct answer survives cross-examination while the idiosyncratic wrong ones get talked down — and the group converges. Pick a puzzle, choose how many agents debate (2–4), then step through the rounds and watch the disagreement shrink and the answer settle onto the truth — while a lone single-model baseline stays stuck on its first wrong guess. Everything is scripted and deterministic — no live model — so the trajectory is identical every run and mirrors the real debate pattern.

🔑 No single agent got smarter — the disagreement did the work. Every instance is the same model at the same settings; the only thing that changed is that they could see and challenge each other. Because their errors are idiosyncratic (each wrong for its own reason) while the correct answer has a reason that holds up to cross-examination, revision pushes the outliers toward the defensible answer and pulls the group's agreement up alongside its accuracy. The lone baseline never gets that second opinion, so its first confident mistake is also its final one. Debate is not free — it costs agents × rounds model calls — and it can fail by groupthink if the agents are too alike or a wrong answer sounds too persuasive. Use it where a wrong answer is expensive and a single pass is unreliable.

Several instances answer → each critiques the others → they revise and converge. Majority vote or a judge reads the final round. More agents and more rounds cost more calls; too-similar agents risk groupthink.

🔨 Build it yourself

    
Multi-Agent Debate is a small orchestration loop around one model: ask it the same question a few times to get independent answers, then feed each answer the others' answers and ask it to critique and revise — repeat for a couple of rounds — and finish by aggregating. No new model, no fine-tuning; the whole technique is prompt-plumbing plus a vote. Here is the entire thing.

    
    
      
  1. 
        
    1
    Spin up N independent agents and collect first answers
    
        
    Ask the same model the same question N separate times (a small temperature keeps the answers from being identical). These independent first passes are your debaters — and, importantly, they will not all agree.
    
        
    // 1. N independent instances answer the SAME question.
const question = 'How many times does "r" appear in "strawberry"?';
const N = 3;
let answers = await Promise.all(
  Array.from({ length: N }, () =>
    llm(question, { temperature: 0.7 }))   // some will disagree
);
// answers -> ["2", "3", "2"]  (majority is wrong to start)
    
      
    2. 
      
  2. 
        
    2
    Build the debate prompt — show each agent the others
    
        
    The whole trick is the context you hand back in. For each agent, concatenate the other agents' current answers and ask it to weigh them. This cross-talk is what separates debate from just sampling N times.
    
        
    // 2. Each agent sees the OTHERS' answers (not its own).
function debatePrompt(question, mine, others) {
  return `${question}

Other agents answered:
${others.map((a, i) => `Agent ${i + 1}: ${a}`).join("\n")}

Critique their reasoning and yours, then give your
best answer. Change it if they convince you.`;
}
    
      
    3. 
      
  3. 
        
    3
    Run a critique-and-revise round
    
        
    Every agent re-answers with the others in view. A wrong outlier that cannot defend its reasoning tends to fold toward the well-argued answer; a correct agent with an explicit argument holds firm. That asymmetry is the engine.
    
        
    // 3. One round: everyone revises given the others.
async function round(question, answers) {
  return Promise.all(answers.map((mine, i) => {
    const others = answers.filter((_, j) => j !== i);
    return llm(debatePrompt(question, mine, others));
  }));
}
answers = await round(question, answers);   // ["3","3","2"]
    
      
    4. 
      
  4. 
        
    4
    Loop a few rounds — stop early on agreement
    
        
    Two or three rounds is usually enough; most of the gain lands in round one. Stop as soon as the agents agree (no point paying for more calls once they have converged), and always cap the rounds so a stubborn disagreement cannot run forever.
    
        
    // 4. Repeat, but stop as soon as they agree. Cap the rounds.
const MAX_ROUNDS = 3;
for (let r = 0; r < MAX_ROUNDS; r++) {
  if (new Set(answers).size === 1) break;   // converged -> stop
  answers = await round(question, answers);
}
// ["3","3","3"]  -> unanimous, done in 2 rounds
    
      
    5. 
      
  5. 
        
    5
    Aggregate — majority vote or a judge
    
        
    Read the final round into one answer. Majority vote is cheap and works when the group has converged. A judge — one more model call that reads all final answers and their reasoning — is better on ties and when the best-argued minority is actually right.
    
        
    // 5. Collapse the final round to one answer.
function majorityVote(answers) {
  const c = {};
  answers.forEach(a => c[a] = (c[a] || 0) + 1);
  return Object.entries(c).sort((x, y) => y[1] - x[1])[0][0];
}
const final = majorityVote(answers);            // "3"
// or: const final = await judge(question, answers);  // reads reasoning
    
      
    6. 
      
  6. 
        
    6
    Guard the cost — and against groupthink
    
        
    Debate costs agents × rounds calls, so reserve it for questions where a wrong answer is expensive and one pass is unreliable. Diversify the agents (personas, temperatures, even different models) so they fail differently — identical agents just agree on the same mistake — and never trust it blindly: measure it against a single-pass baseline on a real eval.
    
        
    // 6. Reserve it for hard, high-stakes questions; keep agents diverse.
const calls = N * roundsUsed;      // agents x rounds -> your bill
if (isCheapOrCertain(question)) answerOnce(question);
else debate(question, {
  personas: ["skeptic", "literalist", "step-by-step"], // fail differently
  maxRounds: 3
});
// then: compare vs single-pass on an eval set. groupthink is real.

You learned Multi-Agent Debate (Du et al., 2023): run several independent instances of a model on the same question, then over a few rounds let each read, critique, and revise against the others until they converge — aggregating the final round by majority vote or a judge. It works because different instances make different, idiosyncratic errors while a correct answer has reasoning that survives cross-examination, so agreement rises together with accuracy — and a lone single-model baseline, with no second opinion, stays stuck on its first confident mistake. It is distinct from self-consistency (sample N and vote, but with no cross-talk) and from self-refine (one agent critiques itself, so it shares its own blind spots); debate's diversity of independent critics is the whole point. The cost is real — agents × rounds model calls — and it can fail by groupthink when agents are too alike or a wrong answer argues too well, so keep the debaters diverse, cap the rounds, and measure it against a single pass. 👉 Tomorrow — Day 33: Automatic Prompt Engineer (APE) — stop hand-writing prompts and let a model generate, score, and select the instruction for you.
