🧩 Least-to-Most — Decompose, Then Solve in Order

Leap straight at a multi-step problem and a middle step quietly breaks.

A problem made of several linked steps tempts the model to jump to the final answer in one go. Answering one-shot tries to hold every step in its head at once, slips on a middle one, and the whole answer collapses. Least-to-most first decomposes the problem into an ordered list of easier sub-problems, then solves them in sequence — each sub-answer is fed forward into the next — building up to the correct final answer. Pick a problem and watch both paths run:

🔨 Build a least-to-most solver

    
Take a hard, multi-step problem. First prompt to decompose it into an ordered list of simpler sub-problems. Then loop: solve each sub-problem, accumulating the prior answers as context so every step builds on the last. The final sub-answer is the solution — and you can inspect every step in between.

    
    
      
  1. 1
    Start with the hard, multi-step problem
    // A problem with several linked steps is exactly the kind that
// tempts a model to leap to the final answer and slip mid-way.
const problem =
  "Take the phrase 'machine learning is fun'. Concatenate the " +
  "LAST letter of each word, in order, into one string.";

// Answering in one shot, the model often drops or reorders a
// letter. We will decompose it first, then solve step by step.
    2. 

      
  2. 2
    Stage 1 — prompt to DECOMPOSE
    // Don't solve yet. Ask only for an ORDERED list of simpler
// sub-problems, easiest first, that build up to the solution.
const decomposePrompt = `
You are about to solve a hard problem. First, do NOT solve it.
Break it into an ordered list of simpler sub-problems,
easiest first, where each later step can use earlier answers.
Return one sub-problem per line, numbered.

Problem: "${problem}"
Sub-problems:`;

const planText = await llm(decomposePrompt);
// 1. Last letter of "machine"?
// 2. Last letter of "learning"?
// 3. Last letter of "is"?
// 4. Last letter of "fun"?
// 5. Concatenate the four letters in order.
    3. 

      
  3. 3
    Parse the plan into an ordered list
    // Turn the numbered text into an array you can iterate over.
// Each entry is one easy sub-problem to solve in sequence.
const subProblems = planText
  .split("\n")
  .map(line => line.replace(/^\s*\d+[.)]\s*/, "").trim())
  .filter(Boolean);

// subProblems = [
//   'Last letter of "machine"?',
//   'Last letter of "learning"?',
//   'Last letter of "is"?',
//   'Last letter of "fun"?',
//   'Concatenate the four letters in order.' ]
    4. 

      
  4. 4
    Stage 2 — solve each sub-problem in order
    // Loop the sub-problems. Feed every prior answer FORWARD as
// context, so each step builds on the last instead of guessing.
let history = "";                 // accumulated solved steps
for (const sub of subProblems) {
  const stepPrompt = `
Original problem: ${problem}

Solved so far:
${history || "(nothing yet)"}

Now solve ONLY this next sub-problem. Be terse.
Sub-problem: ${sub}
Answer:`;
  const ans = await llm(stepPrompt);
  history += `- ${sub} -> ${ans}\n`;   // carry it forward
}
    5. 

      
  5. 5
    The last sub-answer is the final answer
    // Because the steps build up, the final step's answer IS the
// solution — and 'history' is a full, inspectable trace.
// history:
//   - Last letter of "machine"? -> e
//   - Last letter of "learning"? -> g
//   - Last letter of "is"? -> s
//   - Last letter of "fun"? -> n
//   - Concatenate the four letters in order. -> "egsn"
//
// final answer = "egsn"   ✓  (every middle step survived)
    6. 

      
  6. 6
    Wrap it as one reusable function
    // The whole pattern is: decompose -> solve-in-order -> last.
async function leastToMost(problem) {
  const plan = await llm(
    `List ordered, easiest-first sub-problems (one per line) ` +
    `to solve, do NOT solve:\n"${problem}"`);
  const subs = plan.split("\n")
    .map(l => l.replace(/^\s*\d+[.)]\s*/, "").trim()).filter(Boolean);

  let history = "", last = "";
  for (const sub of subs) {
    last = await llm(
      `Problem: ${problem}\nSolved so far:\n${history}\n` +
      `Solve ONLY:\n${sub}\nAnswer:`);
    history += `- ${sub} -> ${last}\n`;
  }
  return { subs, history, answer: last };
}
    7. 

      
  7. 7
    Steer decomposition with a few-shot example
    // Show one good decomposition so the split lands at the right
// grain — sub-problems the model can each solve in one easy step.
const decomposePrompt = `
Problem: "Last letter of each word in 'red hot car', joined."
Sub-problems:
1. Last letter of "red"?
2. Last letter of "hot"?
3. Last letter of "car"?
4. Join the three letters in order.

Problem: "${problem}"
Sub-problems:`;
// Few-shot pins the grain; one example is usually enough to fix it.
    8. 

      
  8. 8
    Spend the extra calls only where they pay
    // L2M costs one decompose call + one call per sub-problem.
// Gate it: use it on compositional / multi-step problems; skip
// it on single-step questions where there is nothing to split.
async function solve(problem) {
  if (isMultiStep(problem)) {
    return (await leastToMost(problem)).answer;  // decompose first
  }
  return llm(problem);                           // one step -> direct
}
// ⚠️ Decompose when steps CHAIN; not for "2 + 2" — there is
//    no middle step to drop, so a single call is cheaper.
