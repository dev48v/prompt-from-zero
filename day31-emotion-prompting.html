🎭 Emotion-Prompting — stapling a task-free emotional stimulus onto a prompt (“this is very important to my career”, “take a deep breath and work carefully”) measurably nudges the model to try harder on open-ended work: a free knob some phrases turn, some don't, and one turns the wrong way

Same question, one extra sentence of feeling — and the answer gets more careful. Append an emotional stimulus and watch the effort climb.

Ask a model a question and you get a competent, default-effort answer. Now append one sentence that carries no task information at all — “This is very important to my career,” “Take a deep breath and work on this step by step” — and, on open-ended work, the reply often comes back longer, more careful, better organized. That is EmotionPrompt (Li et al., 2023): an emotional stimulus stapled onto a prompt measurably nudges an LLM to try harder — with no change to the model, the task, or the temperature. It is neither magic nor reliable: some phrases help, polite filler does nothing, and a threat can make things worse. Pick a task, toggle stimuli on the left, and watch the plain answer and the emotionally-charged answer diverge — while the per-stimulus effect bars show which phrases are actually pulling weight. Everything is scripted and deterministic — no live model — so the effect is identical every run and grounded in the paper's real pattern.

🔑 You changed the effort, not the model. The weights, the task, and the settings were identical in both columns — the only difference is a sentence of feeling stapled to the end. Emotional-stakes language is statistically entangled in the training data with careful, high-effort human writing (the things people said mattered were the things people wrote carefully), so the phrase steers generation toward that “try-harder” region. But it is a nudge, not a lever: it helps most on open-ended quality tasks, does nothing on fixed-answer ones, fades on heavily-tuned models, and can backfire when the “emotion” is pressure or threat. Treat it as a cheap knob to A/B test — never as a guarantee.

🔨 Build it yourself

    
Emotion-Prompting is the smallest possible technique: one extra sentence. There is no loop, no tool, no second model — you concatenate a stimulus onto your prompt and check whether it helped. The whole craft is in choosing the right stimulus for the task, and proving it actually moves your metric instead of trusting a screenshot. Here is the entire thing.

    
    
      
  1. 
        
    1
    Start with a clean base prompt
    
        
    Write the task the normal way, with no tricks. This is your control — the answer everything else is measured against. If you cannot state the plain prompt clearly, no emotional suffix will save it.
    
        
    // 1. A normal, clean prompt -- no tricks yet.
const task = "Draft a friendly follow-up email to a client who went quiet.";
let prompt = task;
// this is your control. every variant is measured against it.
    
      
    2. 
      
  2. 
        
    2
    Append an emotional stimulus — that is the whole trick
    
        
    Concatenate one task-free, feeling-loaded sentence onto the prompt. No loop, no tool, no second model — literally one line of string concatenation. That is the entire technique.
    
        
    // 2. Emotion-Prompting IS this line: concatenate a stimulus.
const STIMULUS  = " This is very important to my career.";
const emoPrompt = task + STIMULUS;        // that is the whole method
const answer    = await llm(emoPrompt);
// one extra sentence -- no loop, no tool, no 2nd model.
    
      
    3. 
      
  3. 
        
    3
    Match the stimulus family to the task
    
        
    The paper's stimuli fall into three families — social-esteem (stakes, others' judgment), urgency (this matters, get it right), and self-monitoring (check yourself, step by step). Reach for esteem/stakes on open-ended writing; reach for self-monitoring on reasoning — “step by step” doubles as a reasoning cue.
    
        
    // 3. Match the stimulus family to the task.
const ESTEEM  = " This really matters to me and my team.";              // stakes
const MONITOR = " Take a deep breath and work through it step by step."; // self-check
const CHECK   = " Are you sure? Review your answer before finalizing.";  // self-monitor
// open-ended writing -> esteem/stakes; reasoning -> self-monitoring.
    
      
    4. 
      
  4. 
        
    4
    A/B test it — measure, never assume
    
        
    The effect is small and unstable, so a single good screenshot proves nothing. Run the plain prompt and the stimulus prompt over a held-out eval set and keep the stimulus only if it moves your real metric by more than noise. This is the step that separates a technique from a superstition.
    
        
    // 4. Prove it. Never trust a single example.
const base = await runEval(prompt,            testset);
const emo  = await runEval(prompt + STIMULUS, testset);
console.log("lift:", emo.score - base.score);   // keep only if > noise
    
      
    5. 
      
  5. 
        
    5
    Prefer encouragement to threats; do not over-stack
    
        
    Encouragement and self-monitoring tend to help; threats are a coin-flip that can degrade quality into terse, hedged, blame-avoiding output. And stacking five stimuli does not add up — returns diminish and a bloated emotional preamble starts to distract. One well-chosen line usually beats a pile of them.
    
        
    // 5. Prefer encouragement; resist stacking.
const good = task + " Believe in your abilities and be thorough.";  // tends to help
const bad  = task + " Get this wrong and there are consequences.";  // may backfire
// 5 stacked stimuli => diminishing returns + distracting preamble.
// one well-chosen line usually beats a pile of them.
    
      
    6. 
      
  6. 
        
    6
    Know when to skip it — and what to reach for instead
    
        
    When the answer is fixed and singular — arithmetic, a lookup, a format conversion — there is no “harder” to try, so the stimulus only adds words. Save emotion-prompting for tasks with real quality headroom; on deterministic work, reach for a tool or calculator instead. And on a strong, heavily-tuned model, re-measure: the lift may already be gone.
    
        
    // 6. Skip it when the answer is fixed -- reach for the right tool.
if (isDeterministic(task)) {
  useToolOrCalculator(task);      // 17*24 is 408 regardless of feelings
} else {
  usePersonaPlusEmotion(task);    // quality tasks: steer, then measure
}

🎉 Day 31 of PromptFromZero done.
You learned Emotion-Prompting (EmotionPrompt, Li et al. 2023): append a task-free emotional stimulus — “this is important to my career,” “take a deep breath and work step by step” — and an LLM measurably tries harder on open-ended, quality-graded tasks. It works by steering generation toward the region of the training data where stakes co-occurred with careful human writing — not by giving the model feelings. Three stimulus families (social-esteem · urgency/stakes · self-monitoring); it shines on generative and subjective work and does nothing on deterministic answers; encouragement beats threats (which can backfire); and the whole effect is fragile — small, phrasing-sensitive, and fading on strong RLHF'd models — so you measure it on a held-out eval and pin any winner to a regression test rather than trusting a screenshot.
