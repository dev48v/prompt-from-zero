🔬 Chain-of-Verification — Make the Model Fact-Check Itself

A confident draft can still be wrong. Make it check its own facts before answering.

Language models sound certain even when a fact slips. Chain-of-Verification (CoVe) turns that one risky answer into four disciplined moves: draft a baseline answer, plan one verification question per claim, answer each question independently (so the model isn't anchored to its own draft), then revise — dropping or fixing whatever failed. Watch it catch its own hallucination:

🔨 Wire up a Chain-of-Verification loop

    
Draft a baseline answer, generate one verification question per claim, answer each question in a fresh context so the draft can't bias it, then regenerate the answer using only the verified facts.

    
    
      
  1. 1
    Get the baseline draft answer
    const question = "Name 5 politicians born in New York City.";

// Stage 1 — just answer normally. This draft is fast but may
// contain a slipped fact. We will NOT trust it as final.
const draft = await llm(`${question}
Answer with a short numbered list, one name per line.`);
// e.g. 1. Theodore Roosevelt  2. Franklin D. Roosevelt
//      3. Donald Trump        4. Hillary Clinton  ← born in Illinois!
//      5. Jacob Javits
    2. 

      
  2. 2
    Plan one verification question per claim
    // Stage 2 — turn each factual claim into a check.
const planPrompt = `Here is a draft answer:
${draft}

For EACH claim in the draft, write one short, standalone
verification question that checks just that fact.
Return a JSON array of questions, nothing else.`;

const questions = JSON.parse(await llm(planPrompt));
// ["Was Theodore Roosevelt born in New York City?",
//  "Was Hillary Clinton born in New York City?", ...]
    3. 

      
  3. 3
    Answer each question in a FRESH context
    // Stage 3 — the heart of CoVe. Answer each question on its
// OWN, with NO draft in the prompt. Independence is the point:
// if the model can't see its draft, it can't just repeat the
// error — it judges the fact from scratch.
async function verify(q) {
  return llm(`${q}
Answer "TRUE" or "FALSE", then one line of justification.`);
  // ← separate call, separate context, no draft attached
}
const checks = await Promise.all(questions.map(verify));
    4. 

      
  4. 4
    Collect the verdicts (factored verification)
    // Pair each claim with its independent verdict.
const verified = questions.map((q, i) => ({
  question: q,
  passed:   /^\s*TRUE/i.test(checks[i]),
  note:     checks[i]
}));
// One comes back FALSE: "Hillary Clinton was born in
// Chicago, Illinois — not New York City."
    5. 

      
  5. 5
    Regenerate the answer from verified facts only
    // Stage 4 — revise. Hand the model the original question, the
// draft, AND the verification results, and tell it to drop or
// replace any claim that failed.
const finalPrompt = `Question: ${question}
Draft answer:
${draft}

Verification results:
${verified.map(v => `- ${v.question} → ${v.passed ? "OK" : "FAILED: " + v.note}`).join("\n")}

Rewrite the answer. Remove or correct every claim that FAILED
verification. Keep only facts that passed.`;

const finalAnswer = await llm(finalPrompt);
    6. 

      
  6. 6
    Wrap the whole loop as one helper
    async function chainOfVerification(question) {
  const draft     = await llm(`${question}\nNumbered list.`);
  const questions = JSON.parse(await llm(planQuestions(draft)));
  const checks    = await Promise.all(            // independent!
    questions.map(q => llm(`${q}\nTRUE or FALSE + why.`)));
  const verified  = zipVerdicts(questions, checks);
  return llm(reviseFromVerified(question, draft, verified));
}
// draft → plan → verify (fresh) → revise — one call away.
    7. 

      
  7. 7
    Keep verifications independent — don't leak the draft
    // ❌ DON'T: pass the draft into the verify call —
//    the model just agrees with itself (joint, anchored).
// verify(`Draft said X. Is X true?`)   // confirmation bias

// ✅ DO: ask the bare fact, no draft attached (factored).
verify(`Was Hillary Clinton born in New York City?`)

// Factored = one fresh call per claim → catches more errors.
// Joint     = cheaper, fewer calls → but anchors to the draft.
    8. 

      
  8. 8
    Spend the budget where it pays off
    // CoVe costs several calls: 1 draft + N checks + 1 revise.
// Use it where facts matter and errors are cheap to check:
//   ✅ lists of entities, dates, names, citations, specs
//   ✅ "who/when/where" answers, comparison tables
// Skip it where verification is as hard as the task:
//   ❌ open-ended opinions, creative writing, vague summaries
